# Boterator

**Boterator** — это комплексное решение (Telegram-бот + Telegram Web App) для управления платными подписками в закрытых каналах и группах Telegram. 

Проект разработан для автоматизации приема платежей, управления доступом пользователей (принятие заявок, исключение при неоплате), а также предоставляет удобную CRM-систему для администраторов прямо внутри Telegram.

## Основной функционал

- **Telegram Web App (TWA):**
  - Витрина тарифов для клиентов с возможностью оплаты.
  - CRM-панель для администраторов (статистика, управление тарифами и пользователями, рассылки).
- **Автоматизация доступа:**
  - Автоматическое одобрение заявок (Chat Join Requests) после оплаты.
  - Исключение пользователей после истечения подписки (с настраиваемым грейс-периодом).
  - Поддержка "вечных" пользователей (тех, кто был в канале до добавления бота).
- **Интеграция платежей:**
  - Поддержка ЮKassa, Сбербанк и ЮMoney.
  - Mock-режим для безопасного тестирования логики без реальных транзакций.
- **Отказоустойчивость (Graceful Degradation):**
  - Бот продолжает работать в ограниченном режиме, даже если отвалилась база данных или не настроены платежные шлюзы, информируя администратора через логи.

## Технологический стек

- **Бэкенд:** Python 3, FastAPI
- **Бот:** Aiogram 3.x
- **База данных:** MySQL (асинхронное взаимодействие через `aiomysql` и `SQLAlchemy`)
- **Фронтенд:** Jinja2, HTML/CSS/JS (Vanilla)
- **Деплой:** systemd, Nginx (как reverse-proxy для SSL) на Ubuntu.
